## Refactor hotspot inventory

### 1. Framer Motion usage

#### Route transitions

- `src/App.jsx` uses `AnimatePresence` around routed page transitions.
- `src/components/shared/PageWrapper.jsx` wraps route content in motion primitives.
- `src/components/pages/Blog.jsx`, `src/components/pages/Contact.jsx`,
  `src/components/pages/Projects.jsx`, `src/components/pages/Resume.jsx`,
  `src/components/pages/Playground.jsx`, `src/components/pages/Games.jsx`, and
  `src/components/pages/NotFound.jsx` all import Framer Motion for route/page-level animation.

#### Overlays and shell interactions

- `src/components/shared/Modal.jsx` animates reusable dialog entry/exit.
- `src/components/shared/CommandPalette.jsx` animates backdrop + palette container.
- `src/components/shared/Chatbot.jsx` animates the floating launcher and lazy-loaded dialogs.
- `src/components/shared/TerminalMode.jsx` and `src/components/shared/ScrollToTop.jsx`
  animate persistent chrome / utility affordances.
- `src/components/layout/Navbar.jsx` animates nav entrance and mobile menu behavior.

#### Decorative animation

- `src/components/home/Hero.jsx` and `src/components/home/HeroBackground.jsx` animate the
  landing hero and its background shapes.
- `src/components/shared/ScrollReveal.jsx`, `src/components/shared/SkeletonLoader.jsx`,
  `src/components/shared/TechStackVisual.jsx`, `src/components/shared/CustomCursor.jsx`, and
  `src/components/shared/RoastInterface.jsx` use motion for reveal, cursor, and feedback effects.

#### Game interactions

- `src/components/games/TicTacToe.jsx`, `src/components/games/LightsOut.jsx`,
  `src/components/games/MemoryMatch.jsx`, `src/components/games/Minesweeper.jsx`,
  `src/components/games/SimonSays.jsx`, `src/components/games/WhackAMole.jsx`, and
  `src/components/games/SnakeGame.jsx` all import Framer Motion.
- `src/components/pages/Games.jsx` also uses animated route/page transitions around the game hub.

### 2. Custom modal / overlay logic

#### Shared modal foundation

- `src/components/shared/Modal.jsx`
  - Custom-managed behaviors today: focus trapping, Escape handling, backdrop-click dismissal,
    body scroll lock, close-button focus fallback, and animated entry/exit.
  - Used as the shared dialog shell for Playground preview/runner flows.

#### Settings overlay

- `src/components/shared/SettingsModal.jsx`
  - Reimplements modal shell markup instead of composing `Modal.jsx`.
  - Uses `useFocusTrap`, custom backdrop-click dismissal, and explicit focus restoration to
    `#settings-open-btn` on close.
  - Does **not** currently use the shared animated modal shell.

#### Focus-trap infrastructure

- `src/components/shared/useFocusTrap.js`
  - Handles initial focus, previous-focus restoration, Escape-to-close, Tab-looping,
    and optional body scroll locking.
  - This hook is reused by `Modal.jsx`, `SettingsModal.jsx`, `Navbar.jsx`,
    `CommandPalette.jsx`, and `ChatInterface.jsx`.

#### Other overlay-like custom surfaces

- `src/components/shared/CommandPalette.jsx` is effectively a custom dialog with its own
  backdrop, focus trapping, keyboard navigation, and delayed action execution.
- `src/components/shared/Chatbot.jsx` manages floating-dialog lifecycle, outside-click
  detection, `aria-hidden` toggling on main content, and focus restoration.
- `src/components/layout/Navbar.jsx` uses the shared focus trap for the mobile nav drawer,
  which is another overlay-adjacent interaction pattern.

### 3. `useEffect` / async / browser-API-driven work

#### Route and layout effects

- `src/App.jsx` scroll-restores on pathname changes via `window.scrollTo(0, 0)`.
- `src/components/layout/Layout.jsx` reads motion/contrast/pointer media queries,
  persists cursor preferences, and coordinates global keyboard custom events.
- `src/components/layout/Navbar.jsx` listens to scroll state, manages `aria-hidden`,
  and coordinates custom close events for the mobile menu.
- `src/components/shared/ScrollToTop.jsx` adds/removes scroll listeners and performs smooth scroll.

#### Overlay and dialog effects

- `src/components/shared/useFocusTrap.js` centralizes focus + scroll side effects.
- `src/components/shared/CommandPalette.jsx` scrolls the active command into view.
- `src/components/shared/Chatbot.jsx` binds document-level open/close events,
  outside-click handling, and focus restoration for floating dialogs.
- `src/components/shared/SettingsModal.jsx` uses an effect solely for delayed trigger refocus.

#### Async work / network / third-party loading

- No direct `fetch()` calls were found in routed UI components during static inspection.
- `src/components/shared/ChatInterface.jsx` loads/persists chat state through `localStorage`,
  auto-scrolls messages, and triggers `chatWithGemini(...)` async work.
- `src/services/ai.js` performs Gemini API requests and uses persistent rate-limit state,
  making it a likely target for isolation from render paths.
- `src/components/shared/pyodideLoader.js`, `src/components/shared/PythonRunner.jsx`, and
  `src/components/pages/Playground.jsx` dynamically load and prewarm Pyodide, which is likely
  a major contributor to future interaction and bundle work.
- `src/components/pages/Playground.jsx` also uses `useEffect` to write iframe preview content,
  making the route a good baseline candidate for measuring interactive cost.

#### High-noise `useEffect` areas worth simplifying later

- `src/components/shared/ChatInterface.jsx`
- `src/components/shared/Chatbot.jsx`
- `src/components/layout/Layout.jsx`
- `src/components/layout/Navbar.jsx`
- `src/components/pages/Playground.jsx`

## Top 5 highest-value refactor targets

1. `src/App.jsx` + `src/components/shared/PageWrapper.jsx` — current route animation baseline
   should be simplified before introducing view transitions.
2. `src/components/shared/Modal.jsx` + `src/components/shared/SettingsModal.jsx` — duplicate
   overlay behavior suggests an obvious consolidation target.
3. `src/components/shared/useFocusTrap.js` — central hook already owns critical dialog behavior;
   refining it unlocks multiple overlay improvements at once.
4. `src/components/shared/Chatbot.jsx` + `src/components/shared/ChatInterface.jsx` — heavy
   event/focus/effect logic makes this a high-value maintainability and interaction-quality target.
5. `src/components/pages/Playground.jsx` + `src/components/shared/pyodideLoader.js` — strongest
   candidate for runtime-cost reduction, lazy-loading discipline, and route-level performance wins.
